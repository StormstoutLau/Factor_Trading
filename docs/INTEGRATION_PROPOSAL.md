# 多Agent投研框架融合方案

> 整合 Crucix 开源情报分析系统 + 自适应Agent学习机制

---

## 一、问题诊断

### 问题1：缺少非结构化数据分析能力

**现状：**
- 当前系统仅支持结构化因子数据（PE、PB、收益率等）
- 无法处理新闻、公告、社交媒体、研报等非结构化文本
- 缺少事件驱动型交易策略
- 无法捕捉市场情绪的突然变化

**影响：**
- 策略信号滞后于市场信息
- 无法响应突发事件（如政策变化、公司公告）
- 在极端行情下缺乏预警机制

### 问题2：Agent风格硬编码，缺乏适应性

**现状：**
- 价值型Agent永远选低PE股票
- 动量型Agent永远追涨
- 无法根据市场环境调整策略
- 新策略需要重写代码

**影响：**
- 策略在不适合的市场regime下持续亏损
- 无法从回测表现中学习改进
- Agent之间缺乏协同进化

## 二、Crucix 系统可借鉴的核心设计

### 2.1 分层采集架构

```
Crucix 设计                    适配到量化场景
─────────────────────────────────────────────────
RSS 采集层      →    财经新闻API（Tushare、AkShare）
Telegram 采集   →    社交媒体（雪球、微博、东方财富）
GDELT 事件库    →    公司公告（巨潮资讯）
暗网监控        →    研报数据（Wind、Choice）
```

### 2.2 智能融合机制

| Crucix 特性 | 量化适配 |
|------------|---------|
| 多源数据去重 | 同一事件多新闻报道，去重后只计一次 |
| 时间窗口对齐 | 新闻发布时间 vs 交易时间对齐 |
| 置信度加权 | 权威来源（公告）> 媒体报道 > 社交媒体 |

### 2.3 Delta 检测机制

```
Crucix: 检测情报突变（如某地区冲突升级）
  ↓
量化适配: 检测情感得分突变（如某公司负面新闻集中爆发）
  ↓
触发: 自动调仓或告警
```

### 2.4 三级告警机制

| 级别 | Crucix | 量化场景 |
|------|--------|---------|
| FLASH | 军事冲突爆发 | 重大负面新闻（财务造假、高管被捕） |
| PRIORITY | 政策变化 | 业绩不及预期、行业监管 |
| ROUTINE | 常规情报更新 | 日常新闻、研报评级调整 |

## 五、自适应Agent设计详解

### 5.1 从硬编码到学习的转变

**硬编码方式（当前）：**
```python
class ValueAgent(BaseAgent):
    def generate_signals(self, ...):
        # 永远选低PE
        return factor_scores.nsmallest(20)
```

**自适应方式（新设计）：**
```python
class AdaptiveAgent:
    def __init__(self):
        self.learning_engine = BayesianLearningEngine()
        # 注册多个心智模型
        self.learning_engine.register_model('均值回归')
        self.learning_engine.register_model('趋势跟踪')
        self.learning_engine.register_model('动量效应')
    
    def generate_signals(self, ...):
        # 根据当前最优模型的权重动态组合
        model_weights = self.learning_engine.get_model_weights()
        # 例如: {'均值回归': 0.6, '趋势跟踪': 0.3, '动量效应': 0.1}
        
        composite_signal = sum(
            self._apply_model(name, data) * weight
            for name, weight in model_weights.items()
        )
        return composite_signal
    
    def adapt_to_regime(self, regime, historical_performance):
        # 根据回测表现更新模型信念
        for model, performance in historical_performance.items():
            self.learning_engine.evaluate_prediction(
                model, actual_outcome=(performance > 0)
            )
```

### 5.2 五层认知模型

```
┌─────────────────────────────────────────┐
│ 1. 表达DNA                              │
│    • 单股最大权重: 5% → 10%（动态调整）  │
│    • 行业偏好: None → 科技（学习得出）   │
│    • 市值偏好: None → 小盘（牛市时）     │
├─────────────────────────────────────────┤
│ 2. 心智模型（贝叶斯竞争）                │
│    • 均值回归: 置信度 0.6（震荡市有效）  │
│    • 趋势跟踪: 置信度 0.3（趋势市有效）  │
│    • 动量效应: 置信度 0.1（当前无效）    │
├─────────────────────────────────────────┤
│ 3. 决策启发式（从数据中提取）             │
│    • "RSI<30 且 情感得分>0.5 → 买入"    │
│    • "公告前3天 + 负面新闻 → 减仓"       │
├─────────────────────────────────────────┤
│ 4. 反模式（从失败中学习）                │
│    • "不追涨超过5%的股票"               │
│    • "不持有停牌超过3天的股票"          │
│    • "财报前不建仓"（从亏损中学习）       │
├─────────────────────────────────────────┤
│ 5. 诚实边界（局限性认知）                │
│    • "无法预测黑天鹅事件"               │
│    • "流动性危机时模型失效"             │
│    • "对小盘股预测置信度较低"            │
└─────────────────────────────────────────┘
```

### 5.3 贝叶斯学习闭环

```
回测开始
    │
    ├──→ Agent生成信号（基于当前模型权重）
    │
    ├──→ 回测引擎执行交易
    │
    ├──→ 计算绩效（夏普比率、最大回撤等）
    │
    ├──→ 评估各模型的预测准确性
    │       ├── 均值回归模型: 预测正确 ✓
    │       ├── 趋势跟踪模型: 预测错误 ✗
    │       └── 动量效应模型: 预测正确 ✓
    │
    ├──→ 贝叶斯更新模型信念
    │       ├── 均值回归: 后验概率 0.5 → 0.65
    │       ├── 趋势跟踪: 后验概率 0.3 → 0.20
    │       └── 动量效应: 后验概率 0.2 → 0.30
    │
    └──→ 调整表达DNA
            ├── 提高均值回归权重
            └── 降低趋势跟踪权重
    │
    └──→ 下一轮回测（权重已更新）
```

## 八、关键技术决策

### 8.1 情感分析：规则 vs LLM

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 规则+词典 | 快速、可控、成本低 | 泛化能力弱 | Phase 1使用 |
| LLM（GPT） | 理解能力强、泛化好 | 成本高、延迟大 | Phase 2升级 |
| 混合方案 | 规则过滤+LLM精分析 | 复杂度中等 | **最终方案** |

### 8.2 学习机制：在线 vs 离线

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 离线学习 | 稳定、可复现 | 响应慢 | 月度更新 |
| 在线学习 | 实时适应 | 可能过拟合 | 日度微调 |
| 混合方案 | 离线训练+在线微调 | 复杂度中等 | **最终方案** |

### 8.3 信号融合：固定权重 vs 动态权重

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 固定权重 | 简单、稳定 | 无法适应市场 | 基准方案 |
| 动态权重 | 灵活、适应性强 | 可能频繁切换 | **最终方案** |
| 自适应权重 | 根据表现自动调整 | 实现复杂 | Phase 2 |
